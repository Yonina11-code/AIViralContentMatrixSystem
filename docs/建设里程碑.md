# 建设里程碑

> AIViralContentMatrixSystem 三阶段建设路线图

---

## 总览

整个系统采用**三阶段渐进式建设策略**，从 MVP 跑通闭环到平台化持续演进。每一阶段都有明确的交付物和验收标准。

```
Phase 1 (Week 1-3)             Phase 2 (Week 4-10)              Phase 3 (Week 11+)
┌─────────────────────┐       ┌─────────────────────┐       ┌─────────────────────┐
│  快速 MVP 闭环       │ ───> │  半自动化稳定运转     │ ───> │  平台化持续演进     │
│                     │       │                     │       │                     │
│ 架构: 优化四层       │       │ 采集: 扩展 B站/知乎   │       │ 资产: 版本管理+组合   │
│ 采集: RSS+搜索引擎   │       │ 情报: 自动选题+聚类   │       │ 学习: Epsilon-Greedy  │
│ 编辑: 3个Agent       │       │ 编辑: 5个Agent       │       │ 分发: N平台+矩阵      │
│ 发布: 公众号         │       │ 学习: 模板自动排序    │       │                     │
│ 工期: 2.5周           │       │                     │       │                     │
└─────────────────────┘       └─────────────────────┘       └─────────────────────┘
```

## Phase 1：MVP 闭环（第 1-3 周）

### 目标
2-3 周内跑通"RSS/搜索引擎采集 → Agent 编辑生成 → 公众号发布"的完整闭环，验证系统可用性。

> 相比原始方案（4-6 周），数据源从 B站/知乎/微博/RSS 精简为 RSS + 搜索引擎 API，Agent 从 10 个合并为 3 个，周期缩短一半。

## Phase 2：半自动化稳定运转（第 4-10 周）

### 目标
实现半自动化内容生产，情报层提供自动选题和聚类分析，编辑层扩展至 5 个 Agent，学习层开始模板自动排序。

> 原始方案 Phase 2 为 12 周，优化后缩短至 7 周。主要是因为情报层不再尝试因果归因，改做统计相关性分析；Agent 从 10+ 合并为 5 个。

## 关键决策点

以下节点需要在实际建设过程中复审决策：

| 决策点 | 时间 | 选项 | 推荐 |
|--------|------|------|------|
| LLM 选型 | Week 1 | Claude / GPT-4o / DeepSeek | 先用 Claude，性价比最优 |
| Agent 框架 | Week 4 | LangGraph / CrewAI / AutoGen | LangGraph，生态最成熟 |
| 爬虫 vs API | Week 2 | 优先使用官方 API | 有 API 绝不用爬虫 |
| 向量库选择 | Week 10 | pgvector / Qdrant / Milvus | pgvector，与 PostgreSQL 集成最佳 |
| 前端框架 | Week 5 | React / Vue / 纯后端 | React + Vite |
| 自学习启动时机 | Week 16 | 数据量 ≥ 200 篇 / ≥ 500 篇 | ≥ 200 篇启动模板排序 |

## 风险与应对

| 风险 | 触发条件 | 应对方案 |
|------|----------|----------|
| 数据源被封 | 采集频次过高触发反爬 | 降频 + 代理池 + 切换备用数据源 |
| API 成本超支 | LLM 调用量超过预算 | 规则初筛 + 小模型替代大模型 + 设置日预算上限 |
| 内容质量不达标 | AI 生成文章同质化严重 | 增加随机温度 + 强制模板多样性 + 人工审核兜底 |
| 公众号限流/封号 | 发布频次过高或内容违规 | 降低发布频次 + 风险编辑前置过滤 + 备用账号 |
| 自学习无效 | 数据量不足/噪声过大 | 回退到规则引擎，人工分析原因 |
